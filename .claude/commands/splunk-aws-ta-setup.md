Install, render, configure, and validate the Splunk Add-on for AWS (Splunk_TA_aws): CloudTrail and GuardDuty via the SQS-based S3 input, AWS Config via aws_config, an IAM-role or access-key account runbook, the aws index, CIM mapping, and ingestion validation. Complements the Data Manager onboarding path.

Read and follow the instructions in skills/splunk-aws-ta-setup/SKILL.md to help the user. If more detail is needed, also read skills/splunk-aws-ta-setup/reference.md.
